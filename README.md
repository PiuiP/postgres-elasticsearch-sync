# PostgreSQL to Elasticsearch Sync Service

Микросервис для автоматической синхронизации данных из PostgreSQL в Elasticsearch. 
Обнаруживает новые и изменённые записи в базе данных и индексирует их в Elasticsearch для быстрого поиска.

## Возможности
- **Автоматическая синхронизация** - новые и изменённые записи попадают в Elasticsearch
- **Настраиваемый интервал** - частота проверки изменений (по умолчанию 30 секунд)
- **Полнотекстовый поиск** - с поддержкой русского языка
- **Отказоустойчивость** - автоматическое переподключение при сбоях
- **Логирование** - подробные логи всех операций
- **Docker-готовность** - лёгкий деплой в контейнере

## Технологии

- **Python 3.10+**
- **SQLAlchemy** 
- **Elasticsearch**
- **Docker**

## Характеристики

- Ресурсы: зависит от объема индексируемых данных
- CPU: Минимальная нагрузка, бóльшую часть времени сервис находится в ожидании (sleep)
- Скорость: (зависит от размера текста)
- Автоматическое переподключение при сбоях Elasticsearch
- Ошибки индексации не останавливают синхронизацию
- Graceful shutdown при получении сигнала остановки
- Все операции логируются для отладки
- Горизонтальное масштабирование через несколько инстансов
- Каждый инстанс может синхронизировать свою таблицу
- Поддержка bulk-операций для больших объемов (можно добавить)

## Структура проекта
elasticsearch-sync/
├── app/
│ ├── main.py # основной цикл синхронизации
│ ├── config.py # конфигурация из переменных окружения
│ ├── models.py # SQLAlchemy модели таблиц
│ ├── sync.py # логика синхронизации
│ ├── elasticsearch_client.py # клиент для работы с ES
│ ├── logs/ # директория для хранения логов

## Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `POSTGRES_DB_URL` | URL подключения к PostgreSQL | |
| `ELASTICSEARCH_URL` | URL Elasticsearch | |
| `SYNC_INTERVAL_SECONDS` | Интервал синхронизации (в секундах) | `30` |
| `LAST_SYNC_FILE` | Файл для хранения времени последней синхронизации | `last_sync.txt` |
