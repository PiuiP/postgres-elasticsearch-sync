# PostgreSQL to Elasticsearch Sync Service

Микросервис для автоматической синхронизации данных из PostgreSQL в Elasticsearch. 
Обнаруживает новые и изменённые записи в базе данных и индексирует их в Elasticsearch для быстрого поиска.

## Возможности
- **Автоматическая синхронизация** - новые и изменённые записи попадают в Elasticsearch
- **Настраиваемый интервал** - частота проверки изменений (по умолчанию 30 секунд)
- **Полнотекстовый поиск** - с поддержкой русского языка
- **Отказоустойчивость** - автоматическое переподключение при сбоях
- **Логирование** - подробные логи всех операций
- **Docker-готовность** - лёгкий деплой в контейнере

## Технологии

- **Python 3.10+**

- **Elasticsearch**

| `POSTGRES_DB_URL` | URL подключения к PostgreSQL | - |
| `ELASTICSEARCH_URL` | URL Elasticsearch | - |

## ⚠️ Важно перед использованием

Сервис жёстко завязан на структуру таблицы `articles` из `models.py`.

Чтобы использовать с вашей таблицей:
1. Переименуйте/дополните поля в модели `Article`
2. Поправьте маппинг в `elasticsearch_client.py` под ваши типы данных
3. Убедитесь, что в таблице есть поле для отслеживания изменений (по умолчанию `updated_at`)

По умолчанию сервис синхронизирует таблицу `articles` со следующей структурой:
- `id` (первичный ключ)
- `title`, `text`, `source`, `pub_date`, `created_at`, `updated_at`

### Изменение названия индексируемой таблицы:

1. В `models.py` измените `__tablename__` в классе `Article`:
   ```python
   class Article(Base):
       __tablename__ = "ваше_название"
   ```
### Изменение структуры таблицы:
1. Приведите поля модели в `models.py` в соответствие с вашей таблицей
2. Обновите маппинг в `elasticsearch_client.py` (особенно типы полей)
3. В методе article_to_doc() в `sync.py` укажите, как преобразовывать ваши поля в документ Elasticsearch

### Минимальные требования к таблице:
1. Первичный ключ id (целочисленный)
2. Поле updated_at для отслеживания изменений (или другое поле, по которому фильтровать — потребуется правка кода)
